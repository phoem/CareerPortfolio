# ADR 0007: Use Evidence-Gap Analysis Before Asking Questions

## Status

Accepted

## Context

Repeatedly asking for information already stored in the repository wastes time and makes resume work frustrating. At the same time, some roles require evidence that is not yet documented.

## Decision

Before asking Jordan questions, analyze the job requirements against the three canonical resumes and the OKF knowledge bundle. Ask only targeted questions for meaningful evidence gaps.

When a gap exists, ask whether Jordan has relevant experience and gather the minimum details needed to document ownership, implementation, technologies, scale, deployment, and outcomes accurately.

## Consequences

### Positive

- Reduces repetitive questioning.
- Makes questions directly relevant to the active role.
- Expands the knowledge base where it has demonstrated value.

### Negative

- Requires a deliberate evidence-mapping step before drafting.
- Poorly documented concepts may still need clarification.
